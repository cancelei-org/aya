# カテゴリ内固定位置機能 - テスト手順

## 実装内容
オプション2「カテゴリ内絶対位置固定」を実装しました。

### 主な変更点

1. **MainCanvas.tsx**
   - `onNodesChange`: カテゴリ内でノードを移動した際に相対位置を自動更新
   - `onNodeDragStop`: ドラッグ終了時にも相対位置を確実に更新

2. **既存の実装活用**
   - `useSpatialCategorization`: 初回カテゴライズ時の相対位置記録（実装済み）
   - `handleCategoryMove`: カテゴリ移動時のメンバーノード同期移動（実装済み）

## テスト手順

### 1. 基本動作確認
1. 開発サーバーを起動
```bash
npm run dev
```

2. ブラウザコンソールを開いて、デバッグログを確認

### 2. カテゴリ内固定位置のテスト

#### A. 初回カテゴライズ
1. システムノードを作成
2. カテゴリノードを作成
3. システムノードをカテゴリ内にドラッグ
   - コンソールに `📍 Recording relative position` が表示されることを確認
   - 相対位置が記録される

#### B. カテゴリ内でのノード移動
1. カテゴリ内のノードを別の位置にドラッグ
2. コンソールログを確認:
   - `📍 Updating relative position for "[ノード名]" in category` が表示
   - `[onNodeDragStop] Updating relative position` も表示
3. 新しい相対位置が保存される

#### C. カテゴリ移動時の固定位置維持
1. カテゴリをドラッグして移動
2. メンバーノードがカテゴリ内の固定位置を維持して一緒に移動することを確認
3. コンソールログ:
   - `📦 Updating category [ID] member positions to fixed relative positions`
   - `📍 "[ノード名]" fixed position` で各ノードの位置計算が表示

#### D. カテゴリリサイズ時の位置維持
1. カテゴリの端をドラッグしてリサイズ
2. メンバーノードの相対位置が変わらないことを確認
3. カテゴリを縮小した場合、ノードが境界外に出ないよう自動調整される

### 3. 期待される動作

✅ **実装済みの動作:**
- カテゴリにノードを入れると、その時点の位置が「固定位置」として記録
- カテゴリ内でノードを移動すると、新しい「固定位置」に更新
- カテゴリを移動すると、全メンバーが固定位置を維持して移動
- カテゴリをリサイズしても、メンバーの相対位置は維持
- カテゴリから外れると相対位置情報がクリア

### 4. デバッグ情報

開発環境では以下のログが出力されます：
- `📍 Recording relative position` - 初回カテゴライズ時
- `📍 Updating relative position` - カテゴリ内移動時
- `📦 Updating category` - カテゴリ移動時
- `🎯 Node dragging over category` - ドラッグ中の視覚フィードバック

## トラブルシューティング

### 相対位置が更新されない場合
1. ブラウザのキャッシュをクリア
2. React Flow の内部状態が同期されているか確認
3. コンソールエラーがないか確認

### ノードが期待通りに移動しない場合
1. `canvasNodes` の状態を確認
2. `relativePosition` プロパティが正しく設定されているか確認
3. カテゴリの `bounds` が正しく更新されているか確認

# 部品互換性確認機能 アップデート要件仕様書

## 概要
現在実装されている高度な3段階互換性チェックシステム（基本互換性→精密チェック→ソフトウェア統合→代替部品提案）の品質向上と機能拡張を行います。既存の包括的なアーキテクチャを活かしつつ、パフォーマンス、精度、保守性の課題を解決します。

## 現状システムの強み
- 電圧・通信・電力・接続情報の多角的検証
- TypeScript完全対応による型安全性
- 段階的重要度判定（Critical/Warning/Info）
- ソフトウェア・ハードウェア統合チェック
- 代替部品自動提案と互換性スコア

## アップデート要件

### 要件1: 接続された部品間のみの互換性チェック
**ユーザーストーリー:** システムユーザーとして、実際に接続されている部品間の互換性のみを確認したいので、部品リスト全体ではなく接続された部品ペアのみをチェック対象にして処理を最適化したい

#### 受入条件
1. WHEN 互換性チェックを実行する THEN 接続情報（Connection[]）に基づいて接続されている部品ペアのみをチェック対象とする
2. WHEN 接続されていない部品が存在する THEN 互換性チェックをスキップし、接続を促すUI警告のみを表示する
3. WHEN 大規模システムでチェックを実行する THEN O(n²)からO(connections)への処理量削減により応答時間が大幅に短縮される

### 要件2: AI検索・動的データソース統合システム
**ユーザーストーリー:** システムユーザーとして、静的データでカバーされていない最新のライブラリ・ハードウェア組み合わせや正確な価格情報を確認したいので、AI検索と外部APIを活用した動的データソース統合システムを使用したい

#### 受入条件
1. WHEN 静的ライブラリマップにない組み合わせが検出される THEN OpenAI APIを使用してWeb検索により互換性情報を収集する
2. WHEN AI検索を実行する THEN 公式ドキュメント、GitHub、技術フォーラムから信頼度付きの結果を取得する
3. WHEN 部品価格の比較が必要な場合 THEN リアルタイムの市場価格APIを参照して正確な価格情報を表示する
4. WHEN 新しいライブラリやハードウェア要件が公開される THEN 外部APIから最新情報を取得して自動でデータベースを更新する
5. WHEN AI検索結果を表示する THEN 信頼度スコア（公式>GitHub>フォーラム）と情報源URLを併記する
6. WHEN 同じ組み合わせを再検索する場合 THEN キャッシュされた結果を24時間保持して重複検索を回避する
7. IF 外部APIが応答しない場合 THEN キャッシュされた最新データにフォールバックして継続動作する
8. WHEN ユーザーが手動確認を求める場合 THEN 「オンラインで確認」ボタンでAI検索を手動実行できる

### 要件3: 未接続部品の警告表示機能
**ユーザーストーリー:** システムユーザーとして、接続されていない部品があることを把握したいので、互換性チェック時に未接続部品の警告をUIに表示してほしい

#### 受入条件
1. WHEN 互換性チェックを実行する THEN 部品リストと接続情報を比較して未接続部品を特定する
2. WHEN 未接続部品が存在する THEN オレンジ色の警告アイコンと「接続されていない部品があります」メッセージを表示する
3. WHEN 警告をクリックする THEN 未接続部品のリストと推奨接続方法を詳細モーダルで表示する
4. WHEN 全ての部品が接続済みの場合 THEN 緑色の「全部品接続済み」確認表示を行う

### 要件4: 接続方向性の適切な判定機能
**ユーザーストーリー:** ハードウェア設計者として、電力供給と通信プロトコルの方向性を正確に理解した互換性チェックを受けたいので、接続種別に応じた適切な方向性判定機能を使用したい

#### 受入条件
1. WHEN 電力接続（power）をチェックする THEN 供給者→消費者の方向性を必須として判定し、逆方向接続は無効として警告する
2. WHEN 双方向通信プロトコル（I2C、SPI、UART）をチェックする THEN 接続方向に関係なく同一プロトコル同士の互換性を確認する
3. WHEN 単方向通信（シリアル出力など）をチェックする THEN 送信者→受信者の方向性を考慮して互換性を判定する
4. WHEN 接続種別が不明な場合 THEN デフォルトで双方向として扱い、ユーザーに接続種別の明確化を促す
5. WHEN 電力供給能力不足が検出される THEN 供給側の出力容量と消費側の必要電力を比較して具体的な不足量を表示する
6. WHEN プロトコルレベル（3.3V I2C vs 5V I2C）の不一致が検出される THEN 電圧レベル変換の必要性を警告し、推奨変換ICを提案する

### 要件5: AI駆動動的部品仕様取得システム
**ユーザーストーリー:** ハードウェア設計者として、手動でのデータベース維持なしに最新の部品仕様を反映した接続管理をしたいので、AI検索による動的部品仕様取得と柔軟なポートシステムを使用したい

#### 受入条件
1. WHEN 新しい部品がシステムに追加される THEN AI検索で部品の電源・通信仕様を自動取得し、適切なポート構成を生成する
2. WHEN 部品仕様をAI検索する THEN 公式データシート、Arduino/GitHub文書から信頼度70%以上の情報を取得する
3. WHEN AI検索で仕様が取得できない場合 THEN 基本テンプレート（汎用マイコン、センサー等）にフォールバックする
4. WHEN 通信ポートの接続数が上限に達する THEN AI取得した仕様に基づき適切なエラーを表示する
5. WHEN 部品仕様が7日以上古い場合 THEN バックグラウンドでAI検索を実行して仕様を更新する
6. WHEN 複数の電圧レベルを持つ部品の場合 THEN 電圧別に個別ポートを生成し、レベル変換の必要性を自動判定する
7. WHEN 同一プロトコルポート（例：I2C）から複数接続がある THEN 1つのポートから複数の接続線が分岐する視覚表示を行う
8. WHEN 部品のポート構成情報を表示する THEN 取得した仕様の信頼度スコアと情報源URLを併記する

現在ノードの接続ポートは４つだ。しかし例えばteensy4.1のように複数のポートを持ち、コミュニケーションポートと電力関係のポートの２種類がある。通信規格の異なるポートがある場合は通信規格の種類ごとに１ポー
  トづつノード上に用意できるげきだ。電力関係のポートも出力と入力の２種類あるだろうし、出力ポートに関しては電圧の差で複数の種類のポートがあることが考えられる。それらの違いに関してもノード上に接続ポートを
  用意すべきだ。種類ごとに１ポートなので例えばI2Cポートを４つのパーツで使用したくて本来はI２Cが４ポートある場合でも表示上は１つのI2Cポートから４つの接続線がでるようにしたい。また、同じポートに接続できる
  接続線の数はコミュニケーションに関してはそのポートの数を最大とし、それ以上に接続しようとしたらエラーが出るようにして互換性チェックをしたい。電力に関しては、ポートの数と関係なく何本でも接続できるように
  したい。できるだけハードウェアの接続情報を具体的に記録しなおかつこのソフトウェアが複雑にならないような範囲で互換性チェックを行いたい。

# Requirements Specification

## Overview

GitHubリポジトリに公開されているファームウェアプロジェクト（Arduino、ESP32、PlatformIO等）を解析し、使用されているライブラリやハードウェア要件を自動的に抽出・可視化する機能です。これにより、ソフトウェア開発者がハードウェア設計を行う際に、既存のファームウェアコードから必要なハードウェアコンポーネントを正確に把握し、適切なハードウェア設計判断を行うことができます。

## Requirements

### 要件1: GitHubリポジトリURL入力と検証

**ユーザーストーリー:** ハードウェアエンジニアとして、GitHubリポジトリのURLを入力して、ファームウェアプロジェクトを解析したい。これにより、既存のコードベースから必要なハードウェア要件を把握できる。

#### 受入基準

1. WHEN ユーザーがGitHubリポジトリのURLを入力フィールドに入力する THEN システムはURL形式の妥当性を即座に検証する
2. WHEN 有効なGitHub URLが入力される THEN 「解析」ボタンが有効化される
3. WHEN 無効なURLが入力される THEN エラーメッセージが表示され、具体的な修正方法が示される
4. IF プライベートリポジトリのURLが入力された THEN 「プライベートリポジトリはサポートされていません」というメッセージを表示する

### 要件2: ファームウェアファイルの自動検出

**ユーザーストーリー:** システム管理者として、GitHubリポジトリから自動的にファームウェア関連ファイルを検出したい。これにより、手動でファイルを指定することなく解析を開始できる。

#### 受入基準

1. WHEN GitHubリポジトリが解析される THEN システムは以下のファームウェア関連ファイルを自動検出する：
   - `.ino` (Arduinoスケッチファイル)
   - `.cpp`, `.c`, `.h` (C/C++ソースファイル)
   - `platformio.ini` (PlatformIO設定ファイル)
   - `library.properties` (Arduinoライブラリ設定)
   - `CMakeLists.txt` (ESP-IDF プロジェクト)
2. WHEN ファームウェアファイルが検出されない THEN 「ファームウェアファイルが見つかりませんでした」というメッセージを表示する
3. WHEN 複数のプラットフォームのファイルが検出される THEN 検出されたプラットフォーム一覧を表示する

### 要件3: Arduinoライブラリの抽出と解析

**ユーザーストーリー:** Arduino開発者として、使用されているArduinoライブラリとその目的を自動的に把握したい。これにより、必要なハードウェアコンポーネントを特定できる。

#### 受入基準

1. WHEN `.ino`や`.cpp`ファイルが解析される THEN システムは`#include`文からライブラリ名を抽出する
2. WHEN ライブラリが検出される THEN 以下の情報が表示される：
   - ライブラリ名
   - 推定される用途（通信、センサー、モーター制御等）
   - 必要なハードウェアコンポーネント
3. WHEN 標準Arduinoライブラリが含まれる THEN サードパーティライブラリのみを重要として分類する
4. IF 未知のライブラリが検出された THEN 「未知のライブラリ」として分類し、手動確認を促す

### 要件4: ESP32/ESP-IDF プロジェクトの解析

**ユーザーストーリー:** ESP32開発者として、ESP-IDFプロジェクトで使用されているコンポーネントとハードウェア要件を把握したい。これにより、ESP32の機能要件を正確に理解できる。

#### 受入基準

1. WHEN `CMakeLists.txt`または`sdkconfig`ファイルが検出される THEN ESP32プロジェクトとして認識する
2. WHEN ESP32プロジェクトが解析される THEN 以下のコンポーネントを検出する：
   - WiFi/Bluetooth使用の有無
   - GPIO使用パターン
   - SPI/I2C/UART使用状況
   - ADC/DAC使用状況
3. WHEN ESP32特有の機能が検出される THEN 対応するハードウェア要件を推定する
4. IF 電源管理機能が検出された THEN バッテリー駆動対応の推奨事項を表示する

### 要件5: PlatformIOプロジェクトの解析

**ユーザーストーリー:** PlatformIO開発者として、プロジェクト設定からターゲットボードとライブラリ依存関係を把握したい。これにより、正確なハードウェア構成を理解できる。

#### 受入基準

1. WHEN `platformio.ini`ファイルが検出される THEN プロジェクト設定を解析する
2. WHEN プロジェクト設定が解析される THEN 以下の情報を抽出する：
   - ターゲットボード（`board`設定）
   - ライブラリ依存関係（`lib_deps`）
   - フレームワーク（`framework`）
   - ビルド設定（`build_flags`）
3. WHEN ターゲットボードが特定される THEN ボード固有のハードウェア仕様を表示する
4. IF 複数のenvが定義されている THEN 環境ごとの要件差分を明示する

### 要件6: ハードウェア要件の推定と表示

**ユーザーストーリー:** ハードウェア設計者として、検出されたライブラリから必要なハードウェアコンポーネントを自動的に推定してほしい。これにより、適切なハードウェア選択を行える。

#### 受入基準

1. WHEN ライブラリが解析される THEN 以下のハードウェア要件を推定する：
   - 必要なセンサー類（DHT22、加速度計等）
   - 通信モジュール（WiFi、Bluetooth、LoRa等）
   - 表示デバイス（LCD、OLED等）
   - アクチュエーター（サーボ、ステッピングモーター等）
2. WHEN ハードウェア要件が推定される THEN 推定根拠となるライブラリ名を併記する
3. WHEN 競合する要件が検出される THEN 警告メッセージを表示する
4. IF 電力要件が推定可能な場合 THEN 推定消費電力と電源推奨事項を表示する

### 要件7: 既存ソフトウェアコンテキストとの統合

**ユーザーストーリー:** システムユーザーとして、既存のソフトウェアコンテキスト機能とファームウェア解析結果を統合して表示したい。これにより、総合的なプロジェクト要件を把握できる。

#### 受入基準

1. WHEN ファームウェア解析が完了する THEN 既存のソフトウェアコンテキスト表示に結果を統合する
2. WHEN 統合表示が行われる THEN ファームウェアライブラリを専用セクションに分類表示する
3. WHEN ハードウェア要件が重複する THEN 重複を統合し、ソース（ソフトウェア/ファームウェア）を明記する
4. IF ソフトウェアとファームウェアで矛盾する要件が検出された THEN 警告を表示し、確認を促す

### 要件8: 解析結果のキャッシュと更新管理

**ユーザーストーリー:** 効率的なシステム利用者として、一度解析したリポジトリの結果をキャッシュして、再解析時の処理時間を短縮したい。

#### 受入基準

1. WHEN リポジトリの解析が完了する THEN 結果をローカルストレージにキャッシュする
2. WHEN 同じリポジトリが再度解析される THEN キャッシュの有効性を確認する
3. WHEN キャッシュが有効（24時間以内）である THEN キャッシュされた結果を即座に表示する
4. IF リポジトリが更新されている可能性がある THEN 「最新情報を取得」オプションを提供する

### 要件9: エラーハンドリングとユーザーフィードバック

**ユーザーストーリー:** システム利用者として、解析中に発生するエラーを適切に理解し、対処方法を知りたい。

#### 受入基準

1. WHEN GitHub APIの制限に達する THEN レート制限の状況と回復予定時刻を表示する
2. WHEN ネットワークエラーが発生する THEN 再試行オプション付きのエラーメッセージを表示する
3. WHEN リポジトリにアクセスできない THEN 具体的な理由（404、プライベート等）を表示する
4. IF 解析処理中にタイムアウトが発生した THEN 部分的な結果があれば表示し、完全な解析の再実行オプションを提供する

### 要件10: パフォーマンスと使いやすさ

**ユーザーストーリー:** 効率的な開発者として、迅速かつ直感的にファームウェア解析を実行したい。

#### 受入基準

1. WHEN 小規模リポジトリ（<100ファイル）が解析される THEN 10秒以内に結果を表示する
2. WHEN 大規模リポジトリが解析される THEN プログレスバーと現在の処理状況を表示する
3. WHEN 解析が実行される THEN バックグラウンドで処理し、他の機能の使用を阻害しない
4. IF 解析に時間がかかる場合 THEN キャンセル機能を提供する

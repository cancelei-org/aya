# Requirements Specification

## Overview
音声制御によるVLM（Vision Language Model）を活用したハードウェアデバッグ支援システム。ユーザーが音声でAIと対話しながら、カメラで見せるハードウェアコンポーネントの問題を診断・デバック方法を提案するリアルタイムサポート機能。既存のHardware Debug Supportに実装したくて、UI上の右のConversationは削除して、チャットパネルにユーザーのプロンプトとVLMの結果を表示したい。（通常のチャットと統一したい）。

## Requirements

### Requirement 1: ハードウェアデバッグ特化型AI対話
**User Story:** ハードウェアエンジニアとして、音声でAIに症状を説明し、カメラで回路を見せながら問題の診断を受けたい。これにより迅速なトラブルシューティングが可能になる。AYAに登録されているハードウェア・ソフトウェアコンテキストとカメラ情報を基にデバック方法を提案する。

#### Acceptance Criteria
1. WHEN ユーザーが「LEDが点灯しない」と話しかけた THEN AIが関連する診断手順を音声で案内する
2. WHEN ユーザーがカメラで回路基板を見せた THEN AIが配線、はんだ付け、部品の向きなどの視覚的問題を検出する
3. IF 画像が不鮮明な場合 THEN AIが「もっと近づけて」「明るくして」などの具体的な撮影指示を出す
4. WHEN デバッグセッションが開始された THEN 専門的なハードウェアデバッグプロンプトが自動的に適用される

### Requirement 2: パフォーマンスとユーザビリティ
**User Story:** エンジニアとして、ストレスなく音声対話とビジュアル診断を行い、効率的にデバッグ作業を進めたい。音声対話は相手が話しかけた言語と同じ言語で返答を行うこと。

#### Acceptance Criteria
1. WHEN 音声入力した THEN 1秒以内に音声認識結果が表示される
2. WHEN 画像を撮影した THEN 2秒以内に視覚的分析が開始される
3. IF ネットワークが不安定な場合 THEN ローカルキャッシュで基本機能が継続できる
4. WHEN 長時間のデバッグセッション中 THEN 自動的に要約と進捗が表示される

システムはシンプルなロジックで実装するべきで、既存のシステムをうまく使うべき。
